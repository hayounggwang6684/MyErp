## 1. 목적

이 문서는 관리자 로컬 전용 UI가 일반 사용자 UI와 같은 디자인 시스템을 공유하면서도 접근 경계와 경고 표시를 분명히 유지하도록 정의한다.

## 2. 기본 원칙

- 관리자 UI는 Electron 일반 사용자 UI와 같은 컴포넌트를 재사용한다.
- 경로는 `/admin/*`으로 분리한다.
- 외형은 동일 계열이지만 `관리자 로컬 전용` 표시를 항상 유지한다.
- 접근 제어는 프런트 라우트 숨김이 아니라 로컬 바인딩과 배포 경계에서 강제한다.
- 사용자 관리, 업데이트 관리, 감사 로그 같은 민감 운영 기능은 관리자 UI에만 배치한다.

## 3. 시각적 구분 방식

- 상단 바에 `관리자 로컬 전용` 배지 고정
- 페이지 헤더에 관리자 전용 경고 라벨 배치
- 민감 기능 버튼은 재인증 아이콘 또는 배지를 함께 표시
- 일반 사용자 UI와 다른 별도 디자인을 만들지 않고 강조 레이어만 추가

## 4. 허용 메뉴 구조

- 운영 현황
- 사용자 관리
- 업데이트 관리
- 감사 로그

메뉴 원칙:

- `직원 관리`는 관리자 로컬 화면에서 별도 메뉴로 만들지 않는다.
- `사용자 관리` 화면은 두 영역으로 나뉜다.
  - `ERP 사용자 계정`: 로그인 계정, 잠금, MFA, 권한 관리
  - `직원 기본정보`: 부서, 직무, 연락처, 업무 상태 관리
- 모든 직원이 ERP 사용자는 아니며, ERP 사용자는 특정 직원 레코드와 연결되는 구조를 기본으로 한다.
- 일반 Windows 클라이언트의 `직원관리` 탭은 운영 조회용으로 유지하되, 관리자 조치 기능은 넣지 않는다.

## 5. 공통 컴포넌트 재사용 규칙

- 버튼, 입력, 테이블, 모달은 일반 UI와 동일 컴포넌트를 사용한다.
- 관리자 화면 때문에 별도 버튼 세트를 만들지 않는다.
- 관리자 전용 차이는 `badge`, `header marker`, `reauth indicator`로 처리한다.

## 6. 민감 기능 패턴

적용 대상 예시:

- 계정 잠금 해제
- 감사 로그 조회

패턴:

1. 기능 버튼 노출
2. 재인증 필요 안내
3. 재인증 성공 후 수행
4. 수행 결과와 감사 기록 안내

## 7. 로컬 전용 안내 문구 기준

- `이 화면은 서버가 설치된 장비에서만 사용할 수 있습니다.`
- `민감 작업은 추가 인증 후 진행됩니다.`
- `모든 관리자 작업은 감사 로그에 기록됩니다.`

## 8. 구현 우선순위

1. `/admin/login` 시각 규칙
2. `/admin/dashboard` 운영 현황 화면
3. `/admin/users` 사용자 관리 화면
4. `/admin/updates` 업데이트 관리 화면
5. `/admin/audit` 감사 로그 화면
