## 1. 목적

이 문서는 관리자 로컬 전용 UI가 일반 사용자 UI와 같은 디자인 시스템을 공유하면서도 접근 경계와 경고 표시를 분명히 유지하도록 정의한다.

## 2. 기본 원칙

- 관리자 UI는 Electron 일반 사용자 UI와 같은 컴포넌트를 재사용한다.
- 경로는 `/admin/*`으로 분리한다.
- 외형은 동일 계열이지만 `관리자 로컬 전용` 표시를 항상 유지한다.
- 접근 제어는 프런트 라우트 숨김이 아니라 로컬 바인딩과 배포 경계에서 강제한다.
- 사용자 관리, 권한 변경, 감사 로그, 세션 강제 종료 같은 기능은 관리자 UI에만 배치한다.

## 3. 시각적 구분 방식

- 상단 바에 `관리자 로컬 전용` 배지 고정
- 페이지 헤더에 관리자 전용 경고 라벨 배치
- 민감 기능 버튼은 재인증 아이콘 또는 배지를 함께 표시
- 일반 사용자 UI와 다른 별도 디자인을 만들지 않고 강조 레이어만 추가

## 4. 허용 메뉴 예시

- 관리자 대시보드
- 사용자 관리
- 권한 관리
- 세션 관리
- 감사 로그
- 보안 예외 승인

## 5. 공통 컴포넌트 재사용 규칙

- 버튼, 입력, 테이블, 모달은 일반 UI와 동일 컴포넌트를 사용한다.
- 관리자 화면 때문에 별도 버튼 세트를 만들지 않는다.
- 관리자 전용 차이는 `badge`, `header marker`, `reauth indicator`로 처리한다.

## 6. 민감 기능 패턴

적용 대상 예시:

- 계정 잠금 해제
- 권한 변경
- 예외 권한 승인
- 세션 강제 종료
- 감사 로그 조회

패턴:

1. 기능 버튼 노출
2. 재인증 필요 안내
3. 재인증 성공 후 수행
4. 수행 결과와 감사 기록 안내

## 7. 로컬 전용 안내 문구 기준

- `이 화면은 서버가 설치된 장비에서만 사용할 수 있습니다.`
- `민감 작업은 추가 인증 후 진행됩니다.`
- `모든 관리자 작업은 감사 로그에 기록됩니다.`

## 8. 구현 우선순위

1. `/admin/login` 시각 규칙
2. `/admin/dashboard` 배지와 로컬 전용 표시
3. 향후 사용자/권한/세션 화면에 동일 패턴 확장
