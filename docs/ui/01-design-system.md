## 1. 목적

이 문서는 ERP 화면에 공통으로 적용할 색상, 타이포, 간격, 컴포넌트 규칙을 정의한다.

## 2. 브랜드 방향

- 회사 로고의 선명한 블루 계열을 메인 컬러로 사용한다.
- 화면 인상은 `정밀`, `신뢰`, `정돈`, `산업 장비 업무용`에 가깝게 유지한다.
- 홍보 사이트 같은 큰 그래픽보다 표, 상태, 숫자, 입력 흐름의 선명함을 우선한다.
- 로그인과 초기 시연 화면의 기준안은 `03 미니멀 코퍼레이트`를 사용한다.
- 기본 룩은 화이트 배경, 절제된 블루 포인트, 얇은 구획선, 정돈된 카드 패턴으로 유지한다.

## 3. 컬러 시스템

권장 기본 색상:

- `brand-700`: `#0F4EA8`
- `brand-600`: `#1A5DB8`
- `brand-500`: `#2A6FD2`
- `brand-100`: `#DCE9FA`
- `ink-900`: `#152033`
- `ink-700`: `#344257`
- `ink-500`: `#637086`
- `line-300`: `#D4DBE6`
- `surface-0`: `#FFFFFF`
- `surface-50`: `#F6F8FB`
- `surface-100`: `#EEF2F7`

상태 색상:

- `success-600`: `#1D7A43`
- `warning-600`: `#A96800`
- `danger-600`: `#B42318`
- `info-600`: `#175CD3`

사용 원칙:

- 주요 CTA, 활성 메뉴, 링크, 선택 상태는 `brand-*`
- 본문 텍스트는 `ink-*`
- 배경은 `surface-*`
- 선과 구획은 `line-*`
- 오류와 경고는 고대비로 명확하게 표시
- 초기 인증 화면은 `surface-0`, `surface-50`, `brand-700` 조합을 기본으로 한다.

## 4. 타이포그래피

권장 방향:

- 한글 가독성이 좋은 산세리프
- 숫자와 표 정렬이 안정적인 폰트 사용

텍스트 계층:

- `Display`: 로그인 타이틀, 큰 요약 수치
- `Heading L`: 페이지 제목
- `Heading M`: 섹션 제목
- `Body`: 기본 본문, 폼 라벨
- `Caption`: 보조 정보, 설명 문구
- `Mono Numeric`: 수량, 금액, 재고 수치

원칙:

- 가독성을 위해 과한 굵기 변화보다 크기와 여백으로 계층 구분
- 표 내부 텍스트는 작되 지나치게 작지 않게 유지
- 숫자, 금액, 수량은 오른쪽 정렬 기준을 고려

## 5. 간격 및 레이아웃

기본 간격 단위:

- `4`, `8`, `12`, `16`, `20`, `24`, `32`

레이아웃 원칙:

- 좌측 `SideNav`, 상단 `TopBar`, 본문 `Content` 3단 구조
- 본문은 카드보다 업무 블록과 테이블 중심
- 밀도 높은 ERP 화면을 위해 카드 간격은 좁고 그룹 경계는 명확하게 유지
- 로그인 화면은 좌측 브랜드 패널과 우측 `clean panel` 인증 카드 구조를 기본으로 한다.

## 6. 공통 컴포넌트 세트

### 6.1 쉘

- `AppShell`
- `TopBar`
- `TabBar`
- `SectionHeader`

### 6.2 버튼

- `PrimaryButton`
- `SecondaryButton`
- `DangerButton`
- `GhostButton`

버튼 규칙:

- 페이지마다 새 버튼 스타일을 만들지 않는다.
- 변형은 `size`, `tone`, `icon-position`, `loading`, `disabled`까지만 허용한다.
- 주요 작업은 `PrimaryButton`, 파괴 작업은 `DangerButton` 고정

### 6.3 입력

- `TextField`
- `PasswordField`
- `OtpField`
- `SelectField`
- `SearchField`

입력 규칙:

- 라벨, 힌트, 오류, 읽기 전용 상태 패턴을 통일한다.
- 폼마다 개별 테두리나 컬러 규칙을 만들지 않는다.
- 초기 로그인, MFA, 관리자 로그인은 같은 입력 패턴과 카드 구조를 재사용한다.
- 조회 화면과 신규 입력 화면은 가능한 같은 필드 배치와 간격을 사용한다. 신규 상태는 빈 값과 입력 가능 상태만 다르게 표시한다.
- 등록 완료 데이터의 단일 필드 수정은 `우클릭 수정` 패턴을 사용한다. 우클릭 컨텍스트 메뉴에서 수정 진입 후 해당 필드만 인라인 편집한다.

### 6.4 데이터 표시

- `StatusBadge`
- `DataTable`
- `InfoCard`
- `SettingsCard`
- `EmptyState`
- `Toast`
- `Modal`
- `ContextMenu`
- `InlineEdit`

### 6.6 탭 규칙

- 상위 업무 이동은 `TabBar` 컴포넌트로 통일한다.
- 탭은 `고객관리 / 주문관리 / 공사관리 / 자산관리 / 부품관리 / 청구관리 / 직원관리 / 설정` 순서를 기준으로 한다.
- `직원관리`는 조건부 렌더링 탭이며, 권한이 아니라 부서 기준으로 노출을 제어한다.
- 활성 탭은 `brand-600` 배경 + 흰색 텍스트를 기본으로 사용한다.
- 비활성 탭은 `surface-0` 배경과 `line-300` 테두리를 사용한다.
- `설정` 탭은 업무 카드가 아니라 `SettingsCard + 입력/토글` 조합을 사용한다.
- 업무 탭 안의 하위 페이지 탭은 상위 `TabBar`와 구분한다. 예를 들어 주문관리의 `주문등록` 하위 탭은 주문관리 내부 workspace 안에서만 동작한다.

### 6.5 상태 배지 규칙

- `ACTIVE`: 파랑 또는 녹색 계열
- `PENDING`: 주황 계열
- `LOCKED`, `ERROR`: 빨강 계열
- `DRAFT`: 회색 계열

## 7. 테이블 원칙

- ERP 핵심 화면은 카드보다 테이블 우선
- 헤더 행은 `surface-100` 배경으로 명확히 구획
- 행 높이는 촘촘하되 클릭 가능 영역은 충분히 확보
- 정렬 가능한 컬럼, 고정 액션 열, 상태 배지 열을 기본 패턴으로 삼는다

권장 기본 컬럼 예시:

- 번호
- 상태
- 고객/거래처
- 품목/모델
- 담당자
- 날짜
- 금액 또는 수량
- 작업

## 8. 상호작용 원칙

- 성공/실패 결과는 즉시 `Toast` 또는 인라인 메시지로 보여준다.
- 파괴적 작업은 `Modal` 확인을 거친다.
- 키보드 입력 흐름을 방해하지 않도록 포커스 스타일을 분명히 둔다.
- 로그인과 업무 입력 화면은 엔터 제출 동작을 지원한다.
- Electron renderer에서는 브라우저 기본 `alert`, `prompt`, `confirm`을 사용하지 않는다. 알림, 확인, 단문 입력은 앱 내부 공통 다이얼로그를 사용한다.
- 버튼은 선택되지 않은 상태에서는 강한 컬러를 쓰지 않고, 선택 또는 주요 실행 상태에서만 브랜드 컬러를 사용한다.
- 모든 버튼은 hover 상태를 가져야 하며, hover로 레이아웃이 흔들리면 안 된다.
- 독립 목록은 각 목록 내부에서 스크롤한다. 전체 앱이나 상위 dashboard shell에 스크롤을 만들지 않는다.
- 컨텍스트 메뉴는 해당 업무 패널 안에서만 기본 우클릭을 대체하고, 전역 contextmenu 차단은 사용하지 않는다.

## 9. 반응형 원칙

- 데스크톱 우선
- 태블릿에서는 사이드바 축소 허용
- 모바일 전면 대응은 현재 우선순위가 아니다

## 10. 금지 규칙

- 페이지별 고유 버튼 스타일 생성 금지
- 장식용 과도한 그라데이션, 유리 효과, 네온 효과 금지
- 업무 상태를 색상만으로 전달하는 방식 금지
- 낮은 대비의 회색 텍스트 남용 금지

## 11. 초기 시연 기준 패턴

- `clean panel`: 밝은 배경 위 얇은 선, 중간 그림자, 둥근 모서리의 인증 카드
- `light surface`: 흰색 또는 매우 연한 회색 면을 기본 작업 배경으로 사용
- `brand accent`: 버튼, 포커스, 활성 메뉴, 강조 문구에만 브랜드 블루를 제한적으로 사용
