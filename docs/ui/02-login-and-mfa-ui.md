## 1. 목적

이 문서는 설치형 클라이언트의 로그인 화면, TOTP MFA 인증 화면, 세션 만료 및 오류 상태 UI를 정의한다.

## 2. 화면 목록

- 클라이언트 로그인 화면
- 클라이언트 MFA 화면
- 세션 만료 화면
- 관리자 로컬 로그인 화면

## 3. 로그인 화면 구조

### 3.1 레이아웃

- 선택 기준안은 `03 미니멀 코퍼레이트`다.
- 좌측: 브랜드 메시지, 회사 소개 한 줄, 지원 업무 키워드
- 우측: `clean panel` 인증 카드
- 전체 배경은 밝은 중성 톤을 유지하고, 좌측 브랜드 영역과 우측 카드 사이에 넉넉한 여백을 둔다.
- 로그인 카드는 화면 중앙 우측에 배치하고 과도한 장식을 피한다.

### 3.2 필수 요소

- 회사 로고
- 서비스명: `엔진 수리 및 부품 판매 ERP`
- 접속 인증서 상태 배지
- `아이디`
- `비밀번호`
- `로그인` 버튼
- 오류 또는 안내 메시지 영역

카드 밀도 기준:

- 라벨과 입력 간격은 촘촘하되 답답하지 않게 유지
- 버튼과 보조 문구는 카드 하단에서 명확히 정렬
- 로그인과 관리자 로그인은 동일 카드 패턴을 사용

### 3.3 인증서 상태 표시

- `접속 인증서 확인 중`
- `접속 인증서 확인 완료`
- `접속 인증서 필요`
- `접속 인증서 오류`

표시 규칙:

- 색상만이 아니라 아이콘과 문구를 함께 사용한다.
- Electron 클라이언트가 직접 인증서를 판단하지 않고 서버가 반환한 확인 결과를 표시한다.
- 인증서 오류가 있으면 서버 응답 기준으로 로그인 버튼 비활성 또는 명확한 오류 반환을 적용한다.

## 4. MFA 인증 화면 구조

### 4.1 목적

- 로그인 1차 통과 후 Authenticator 앱의 6자리 코드를 입력받는다.

### 4.2 필수 요소

- 화면 제목: `2차 인증`
- 안내 문구: 등록된 Authenticator 앱에서 코드를 확인
- `OtpField`
- 만료 또는 실패 메시지 영역
- `인증` 버튼
- `다시 로그인` 버튼
- 로그인 화면과 같은 카드 구조, 같은 버튼 규칙, 같은 여백 체계를 유지한다.
- 앱 시작 시 업데이트 확인 결과를 로그인 화면 또는 설정 패널에서 함께 볼 수 있도록 한다.

### 4.3 상태 패턴

- 정상 입력 대기
- 코드 오류
- 코드 만료
- 허용 횟수 초과 직전 경고
- 계정 잠금 안내

## 5. 오류 상태 패턴

로그인 오류:

- 아이디 또는 비밀번호 불일치
- 인증서 미등록
- 승인대기 계정
- 잠금 계정
- 비활성 계정

MFA 오류:

- 코드 불일치
- 시간 초과
- 재시도 횟수 경고
- 잠금 처리 완료

세션 오류:

- 세션 만료
- 권한 없음
- 재인증 필요

## 6. 메시지 톤

- 짧고 명확한 문장 사용
- 보안 사유는 과도한 내부 정보 없이 전달
- 예:
  - `로그인 정보를 다시 확인해 주세요.`
  - `이 계정은 현재 잠금 상태입니다. 관리자에게 문의해 주세요.`
  - `인증 코드가 올바르지 않습니다.`

## 7. 관리자 로컬 로그인 변형

- `/admin/login`은 일반 로그인 화면과 같은 구조를 사용한다.
- 차이점:
  - 상단 또는 카드 헤더에 `관리자 로컬 전용` 배지 표시
  - 로컬 장비 전용 안내 문구 표시
  - 일반 사용자 경로와 색상 체계는 공유하되 강조 라벨만 다르게 사용
  - 접근 제어는 라우트 자체가 아니라 배포 경계와 로컬 바인딩에서 강제됨을 전제로 한다
  - 시각적으로는 03 시안의 미니멀 톤을 유지하고 경고 배지만 추가한다

## 8. API 연결 기준

- 클라이언트 로그인 -> `POST /api/v1/auth/login`
- 클라이언트 MFA -> `POST /api/v1/auth/mfa/verify`
- 세션 확인 -> `GET /api/v1/sessions/me`
- 로그아웃 -> `POST /api/v1/auth/logout`

## 9. 시연 우선순위

1. 로그인 화면
2. MFA 입력 화면
3. 세션 만료 화면
4. 관리자 로컬 로그인 변형
