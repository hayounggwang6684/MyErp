## 1. 목적

이 문서는 로그인 성공 후 보이는 첫 ERP 대시보드의 공통 레이아웃과 메뉴 구조를 정의한다.

기본 시각 기준은 로그인 화면과 동일한 `03 미니멀 코퍼레이트` 계열이다.

## 2. 라우트

- `/dashboard`
- `/admin/dashboard`

## 3. 전체 레이아웃

```text
+-----------------------------------------------------------+
| TopBar: 로고 | 현재 조직 | 알림 | 사용자 | 로그아웃       |
+----------+------------------------------------------------+
| SideNav  | SectionHeader                                  |
|          | Quick Stats / Alerts / Work Cards / Tables     |
|          |                                                |
|          |                                                |
+----------+------------------------------------------------+
```

## 4. 상단 바

구성:

- 로고
- 현재 화면 제목
- 사용자 이름
- 역할 배지
- 알림 영역
- 로그아웃 버튼

원칙:

- 상단 바는 얇고 정돈된 형태
- 불필요한 큰 배너 금지
- 흰색 또는 매우 옅은 배경 위 얇은 선과 브랜드 블루 포인트만 사용

## 5. 좌측 메뉴

1차 메뉴:

- 대시보드
- 수리
- 부품 판매
- 재고
- 출하
- 청구
- 관리

규칙:

- 아이콘 + 텍스트 병행
- 현재 메뉴는 브랜드 컬러 배경 또는 좌측 액센트로 표시
- 관리자 메뉴는 권한이 없으면 표시하지 않음

## 6. 대시보드 본문 블록

### 6.1 요약 카드

- 오늘 접수 건수
- 진행 중 수리
- 부품 판매 요청
- 재고 경고
- 출하 대기 건수
- 미청구 건수

### 6.2 빠른 작업

- 수리 접수 등록
- 재고 조회
- 주문 조회
- 청구 조회

### 6.3 알림 영역

- 재고 부족
- 승인 대기
- 지연 출하
- 장기 미수

### 6.4 최근 작업 표

권장 컬럼:

- 번호
- 업무 구분
- 상태
- 고객명
- 담당자
- 최근 시각

표 스타일 기준:

- 헤더는 옅은 회색 바탕
- 본문은 흰색 배경과 얇은 행 구분선
- 상태는 배지로 표현하고 수치는 정렬을 명확히 유지

## 7. 정보 밀도 원칙

- ERP 특성상 한 화면에서 상태와 수치를 빠르게 읽을 수 있어야 한다.
- 카드 수는 최소화하고, 실제 작업은 표와 리스트 위주로 구성한다.
- 큰 여백보다 명확한 구획과 정렬을 우선한다.
- 로그인 화면보다 정보 밀도는 높이되 시각 톤은 같은 계열로 유지한다.

## 8. 공통 컴포넌트 사용 규칙

- 레이아웃: `AppShell`, `TopBar`, `SideNav`, `SectionHeader`
- 데이터: `InfoCard`, `StatusBadge`, `DataTable`, `EmptyState`
- 행동: `PrimaryButton`, `SecondaryButton`, `GhostButton`

## 9. 초기 시연 범위

- 메뉴 골격 노출
- 사용자 정보 표시
- 더미 요약 카드
- 더미 최근 작업 테이블
- 로그아웃 버튼 동작 위치 정의
