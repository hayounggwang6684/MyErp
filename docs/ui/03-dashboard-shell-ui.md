## 1. 목적

이 문서는 로그인 성공 후 보이는 첫 ERP 대시보드의 공통 레이아웃과 6개 상위 탭 구조를 정의한다.

기본 시각 기준은 로그인 화면과 동일한 `03 미니멀 코퍼레이트` 계열이다.

## 2. 라우트

- `/dashboard`
- `/admin/dashboard`

## 3. 전체 레이아웃

```text
+------------------------------------------------------------------+
| TopBar: 화면 제목 | 사용자 | 버전 | 업데이트 확인 | 로그아웃      |
+------------------------------------------------------------------+
| TabBar: 고객관리 | 수주관리 | 작업관리 | 재고관리 | 직원관리 | 설정 |
+------------------------------------------------------------------+
| SectionHeader + Quick Actions                                    |
+------------------------------------------------------------------+
| Summary Cards / Recent Table / Personal Settings                 |
+------------------------------------------------------------------+
```

## 4. 상단 바

구성:

- 로고
- 현재 화면 제목
- 사용자 이름
- 역할 배지
- 알림 영역
- 로그아웃 버튼

원칙:

- 상단 바는 얇고 정돈된 형태
- 불필요한 큰 배너 금지
- 흰색 또는 매우 옅은 배경 위 얇은 선과 브랜드 블루 포인트만 사용

## 5. 상위 탭

고정 탭:

- 고객관리
- 수주관리
- 작업관리
- 재고관리
- 설정

조건부 탭:

- 직원관리

규칙:

- 기본 진입 탭은 `수주관리`
- `직원관리`는 로그인 사용자의 `department`가 `관리부`이거나 `STAFF_VIEW` 권한이 있을 때 보인다
- `고객관리`, `수주관리`, `작업관리`, `재고관리`도 각각 역할 기반으로 노출한다
- `설정`은 개인화 전용이며 관리자 보안 기능은 포함하지 않는다
- 탭은 상단 바 바로 아래에 배치하고, 활성 탭은 브랜드 블루 배경으로 표시한다

## 6. 탭별 본문 블록

### 6.1 고객관리

- 요약 카드: 신규 고객, 장기 미응답, 미수 고객, 최근 상담
- 최근 목록: 고객명, 담당자, 최근 상담일, 상태
- 빠른 작업: 고객 등록, 상담 이력, 장기 미응답 조회

### 6.2 수주관리

- 요약 카드: 신규 수주, 견적 진행, 승인 대기, 출하 예정
- 최근 목록: 수주번호, 고객명, 품목/작업, 상태, 담당자
- 빠른 작업: 수주 등록, 견적 조회, 출하 일정 보기

### 6.3 작업관리

- 요약 카드: 작업 접수, 진행 중, 부품 대기, 완료 예정
- 최근 목록: 작업번호, 장비/엔진, 상태, 담당자, 마감 예정
- 빠른 작업: 작업 등록, 일정 보기, 현장 배정 현황

### 6.4 재고관리

- 요약 카드: 재고 경고, 긴급 발주 필요, 입고 대기, 출고 요청
- 최근 목록: 품목명, 현재고, 최소재고, 상태, 위치
- 빠른 작업: 재고 조회, 발주 요청, 입출고 이력

### 6.5 직원관리

- 관리부 직원에게만 노출
- 또는 `STAFF_VIEW` 권한이 있는 사용자에게 노출
- 운영용 직원 정보와 배정 현황만 보여준다
- 포함 항목: 이름, 부서, 직무, 담당 건수, 현재 상태, 연락처
- 제외 항목: 계정 잠금 해제, MFA 초기화, 권한 변경, 관리자 보안 설정

### 6.6 설정

- 자동 로그인 사용 여부
- 로그인 화면 기본 접속 범위 테스트값
- 최근 로그인 아이디 표시 여부
- 현재 앱 버전
- 업데이트 동작 안내

표 스타일 기준:

- 헤더는 옅은 회색 바탕
- 본문은 흰색 배경과 얇은 행 구분선
- 상태는 배지로 표현하고 수치는 정렬을 명확히 유지

## 7. 정보 밀도 원칙

- ERP 특성상 한 화면에서 상태와 수치를 빠르게 읽을 수 있어야 한다.
- 카드 수는 최소화하고, 실제 작업은 표와 리스트 위주로 구성한다.
- 큰 여백보다 명확한 구획과 정렬을 우선한다.
- 로그인 화면보다 정보 밀도는 높이되 시각 톤은 같은 계열로 유지한다.

## 8. 공통 컴포넌트 사용 규칙

- 레이아웃: `TopBar`, `TabBar`, `SectionHeader`
- 데이터: `InfoCard`, `StatusBadge`, `DataTable`, `EmptyState`, `SettingsCard`
- 행동: `PrimaryButton`, `SecondaryButton`, `GhostButton`

## 9. 초기 시연 범위

- 6개 상위 탭 구조 노출
- `직원관리`의 부서 기반 조건부 노출
- 업무 5개 탭의 더미 카드/표/빠른 작업
- `설정` 탭의 실제 개인화 저장
- 로그아웃 버튼 동작 위치 정의
