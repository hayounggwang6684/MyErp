## 1. 목적

이 문서는 접수 이후 수리 진행 절차를 정의한다.

## 2. 업무 개요

- 시작 조건: 수리 접수가 완료되어 작업 대상이 확정된다.
- 종료 조건: 수리 완료, 수리 불가, 재진단 중 하나의 상태로 종료된다.
- 주요 담당자: 수리 엔지니어, 작업 관리자

## 3. 주요 단계

1. 수리 대상 확인
2. 진단 수행
3. 부품 및 작업 내역 기록
4. 수리 완료 처리

## 4. 입력 정보

- 진단 결과
- 작업 담당자
- 사용 부품
- 완료 일시

## 5. 출력 정보

- 수리 결과
- 작업 이력
- 사용 부품 이력
- 완료 또는 보류 상태

## 6. 확인 항목

- 진행 상태 변경 여부
- 수리 결과 기록 여부

## 7. 예외 상황

- 수리 불가 판정 시 고객 안내 및 추가 승인 절차가 필요하다.
- 진단 중 추가 부품이 필요하면 재고 확인 후 대기 상태로 전환한다.
- 동일 증상 재발 시 재수리 또는 클레임 처리 기준을 적용한다.

## 8. 전후 연계 문서

- 접수 시작 단계는 [수리 접수 워크플로우](01-repair-intake-workflow.md)를 따른다.
- 수리에 사용된 자재와 부품 반영은 [재고 워크플로우](05-inventory-workflow.md)와 연결된다.
- 수리 완료 후 비용 청구가 필요하면 [청구서 워크플로우](07-invoice-workflow.md)로 이어진다.

## 9. 향후 보완 항목

- 재수리 프로세스
- 불가 판정 처리 기준
