## 1. 목적

이 문서는 청구서 발행 및 정산 관련 절차를 정의한다.

## 2. 업무 개요

- 시작 조건: 판매, 수리, 주문 등에서 청구 대상이 확정된다.
- 종료 조건: 청구서 발행과 수금 상태 기록이 완료된다.
- 주요 담당자: 회계 담당자, 영업 담당자

## 3. 주요 단계

1. 청구 대상 확인
2. 금액 검증
3. 청구서 발행
4. 수금 상태 반영

## 4. 입력 정보

- 거래처 정보
- 청구 항목
- 발행 일자
- 결제 조건

## 5. 출력 정보

- 청구서 번호
- 발행 문서
- 수금 상태
- 미수금 정보

## 6. 확인 항목

- 금액 일치 여부
- 발행 상태 저장 여부

## 7. 예외 상황

- 청구 금액 불일치 시 원거래 문서를 재검토해야 한다.
- 세금 처리 조건이 다르면 별도 발행 기준을 적용한다.
- 미수금이 장기화되면 회수 관리 단계로 전환한다.

## 8. 전후 연계 문서

- 견적 확정 주문은 [견적 및 주문 워크플로우](03-quotation-order-workflow.md)와 연결된다.
- 부품 판매 청구는 [부품 판매 워크플로우](04-parts-sales-workflow.md)를 참조한다.
- 수리 완료 청구는 [수리 진행 워크플로우](02-repair-process-workflow.md)와 연결된다.

## 9. 향후 보완 항목

- 세금계산 연동
- 미수금 추적
