## 1. 목적

이 문서는 견적서 작성과 주문 전환 절차를 정의한다.

## 2. 업무 개요

- 시작 조건: 고객이 견적 또는 주문 상담을 요청한다.
- 종료 조건: 견적서 발행 또는 주문 등록이 완료된다.
- 주요 담당자: 영업 담당자, 승인 담당자

## 3. 주요 단계

1. 견적 요청 접수
2. 품목 및 금액 산정
3. 견적서 발행
4. 주문 확정 및 주문 등록

## 4. 입력 정보

- 고객 정보
- 견적 품목
- 수량 및 단가
- 유효 기간

## 5. 출력 정보

- 견적번호
- 견적서 문서
- 주문번호
- 주문 상태

## 6. 확인 항목

- 견적 승인 여부
- 주문 전환 여부

## 7. 예외 상황

- 단가 정책 예외가 있으면 승인 절차를 거쳐야 한다.
- 재고 부족 품목은 납기 조정 또는 대체 제안이 필요하다.
- 견적 유효 기간 만료 후 주문 요청 시 재견적이 필요하다.

## 8. 전후 연계 문서

- 주문 가능 재고 확인은 [재고 워크플로우](05-inventory-workflow.md)와 연결된다.
- 주문 확정 후 비용 청구는 [청구서 워크플로우](07-invoice-workflow.md)로 이어질 수 있다.
- 실제 출하가 필요한 주문은 [출하 워크플로우](06-shipping-workflow.md)와 연결한다.

## 9. 향후 보완 항목

- 견적 이력 관리
- 다중 버전 견적 처리
