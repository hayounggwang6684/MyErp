## 1. 목적

이 문서는 입고, 출고, 조정 등 재고 처리 절차를 정의한다.

## 2. 업무 개요

- 시작 조건: 입고, 출고, 이동, 조정 요청이 발생한다.
- 종료 조건: 재고 수량과 이력이 시스템에 반영된다.
- 주요 담당자: 창고 담당자, 구매 담당자, 운영 담당자

## 3. 주요 단계

1. 재고 입고 등록
2. 재고 출고 등록
3. 재고 이동 또는 조정
4. 재고 현황 확인

## 4. 입력 정보

- 품목 정보
- 수량
- 입출고 구분
- 처리 일시

## 5. 출력 정보

- 재고 수량 변경 이력
- 입출고 내역
- 조정 사유 기록
- 현재고 현황

## 6. 확인 항목

- 재고 수량 반영 여부
- 이력 저장 여부

## 7. 예외 상황

- 음수 재고가 발생하면 즉시 등록을 중단하고 원인을 확인한다.
- 실물 재고와 시스템 수량 불일치 시 조정 승인 절차를 거친다.
- 유효기간 또는 시리얼 관리 대상 품목은 별도 추적 기준을 적용한다.

## 8. 전후 연계 문서

- 수리 작업에 사용된 부품 반영은 [수리 진행 워크플로우](02-repair-process-workflow.md)와 연결된다.
- 판매 가능 수량 확인은 [부품 판매 워크플로우](04-parts-sales-workflow.md)와 연결된다.
- 주문 확정 시 출고 가능 여부 검토는 [견적 및 주문 워크플로우](03-quotation-order-workflow.md), [출하 워크플로우](06-shipping-workflow.md)와 함께 본다.
