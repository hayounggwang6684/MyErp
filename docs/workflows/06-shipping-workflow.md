## 1. 목적

이 문서는 출하 및 배송 처리 절차를 정의한다.

## 2. 업무 개요

- 시작 조건: 출하 대상 주문 또는 판매 건이 확정된다.
- 종료 조건: 송장 등록과 배송 상태 반영이 완료된다.
- 주요 담당자: 물류 담당자, 창고 담당자

## 3. 주요 단계

1. 출하 대상 확인
2. 포장 및 출고 준비
3. 송장 정보 등록
4. 배송 상태 갱신

## 4. 입력 정보

- 주문 또는 출하 번호
- 수령인 정보
- 배송 방법
- 송장 번호

## 5. 출력 정보

- 출하 번호
- 송장 정보
- 배송 상태
- 인수 인계 이력

## 6. 확인 항목

- 출하 상태 반영 여부
- 배송 정보 저장 여부

## 7. 예외 상황

- 출하 직전 재고 부족이 확인되면 출하를 중지하고 재고 검증을 다시 수행한다.
- 배송지 정보 오류가 있으면 고객 확인 후 송장 발행을 보류한다.
- 부분 출하가 필요한 경우 잔여 수량을 별도 관리한다.

## 8. 전후 연계 문서

- 주문 기반 출하는 [견적 및 주문 워크플로우](03-quotation-order-workflow.md)와 연결된다.
- 부품 판매 출하는 [부품 판매 워크플로우](04-parts-sales-workflow.md)와 연결된다.
- 실제 출고 수량 차감은 [재고 워크플로우](05-inventory-workflow.md)와 함께 확인한다.

## 9. 향후 보완 항목

- 택배사 연동
- 배송 추적 자동화
