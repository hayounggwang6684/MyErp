## 1. 목적

이 문서는 수리 접수 업무의 표준 처리 절차를 정의한다.

## 2. 업무 개요

- 시작 조건: 고객이 수리 요청을 접수한다.
- 종료 조건: 접수번호가 발급되고 접수 상태가 저장된다.
- 주요 담당자: 접수 담당자, 고객 지원 담당자

## 3. 주요 단계

1. 고객 정보 확인
2. 접수 대상 제품 등록
3. 증상 및 요청 사항 기록
4. 접수 완료 및 접수번호 발급

## 4. 입력 정보

- 고객 정보
- 제품 정보
- 고장 증상
- 접수 일시

## 5. 출력 정보

- 접수번호
- 접수 상태
- 고객 접수 이력
- 후속 수리 요청 데이터

## 6. 확인 항목

- 필수 정보 누락 여부
- 접수 상태 반영 여부

## 7. 예외 상황

- 고객 정보가 없으면 신규 고객 등록 또는 임시 접수 여부를 판단한다.
- 제품 식별 정보가 불충분하면 접수를 보류하고 추가 정보를 요청한다.
- 보증 여부 확인이 필요한 경우 별도 검토 상태로 전환한다.
